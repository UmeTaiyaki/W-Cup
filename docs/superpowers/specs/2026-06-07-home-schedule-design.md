# ホームタブ「試合日程」設計

- 日付: 2026-06-07
- ステータス: 設計確定（実装前）
- 関連: [[wcup-bracket-2026]] / 管理画面の試合日程（`functions/_lib/defaults.js` の `schedule`）

## 1. 目的・スコープ

ホームタブ（現 `SummaryScreen`）を、管理画面で管理されている試合日程を見せる
**直近フォーカス型の試合日程ビュー**に作り替える。

- 既存コンテンツ（優勝サマリー／みんなの優勝予想／優勝予想の分布）は
  予想タブへ移行済みのため **ホームから削除** する。
- データは `window.WC.SCHEDULE`（`GET /api/config` 由来）を読むだけの **読み取り専用**。
  書き込み・予想連動・結果/スコア表示は行わない。

### スコープ外（今回やらない）

- 自分の予想バッジ等の予想連動表示
- 実際の試合結果・スコア表示
- フィルタUI（グループ/決勝T 切替などのB案要素）
- 日付ジャンプ・検索

将来拡張の余地としては残すが、本設計には含めない。

## 2. データ

`window.WC.SCHEDULE` は試合オブジェクトの配列。1件の形:

```js
{ date: '2026-06-12', time: '04:00', round: 'A', a: 'MEX', b: 'RSA', note: 'Mexico City' }
```

- `date` / `time` … JST 変換済み（ISO日付 + HH:MM）
- `round` … グループ記号（A–L）または ラウンド名（R32/R16/QF/SF/3rd/F 等）
- `a` / `b` … グループは3文字コード。ノックアウトは未確定スロット表記
  （`1X`=グループX1位 / `2X`=グループX2位 / `3(...)`=該当群の3位 / `W##`=第##試合勝者 / `L##`=敗者）
- `note` … 開催地（ノックアウトは「#試合番号 開催地」形式が混ざる）

合計104試合（グループ72 + ノックアウト32）。`time` 未設定・`date` 欠落の要素が
混ざっても破綻しないこと。

## 3. 画面構成（上から）

1. **ヘッダー** — ワードマーク + 「ホーム」eyebrow（既存スタイル踏襲）
2. **フォーカス日ヘッダー** — 「📅 6月12日(金) の試合 ／ 1 / N」
3. **ヒーロー = スワイプカルーセル** — フォーカス日の全試合を1枚ずつ表示
   - タッチスワイプ + 左右矢印 + ドットインジケータの併用
     （モバイル=スワイプ主体、デスクトップ=矢印主体）
   - 各カード: 両国旗（大）/ チームコード / KO時刻 / 会場 / グループ・ラウンド章
   - 先頭（または各カード）に開催までの「あと X 日」（当日は「本日」）
4. **タイムライン** — フォーカス日の **翌日以降** を日付ごとにグルーピング
   - 日付見出し（「6月13日(土) ・ N試合」）+ コンパクト行
   - 行: 時刻 / 旗+コード vs コード+旗 / グループ・ラウンド章
   - スクロールで全日程の末尾まで到達できる

フォーカス日自身はヒーローにあるため、タイムラインでは重複表示しない。

## 4. フォーカス日ロジック

`pickFocusDate(dates, today)`（純関数）で決定:

- 今日（JST）に試合があれば **今日**。
- 無ければ **今日より後で最初に試合がある日**（次の試合日へ自動スキップ）。
- 今日以降に試合が無い（全試合終了後）→ **最後の試合日** を表示して破綻させない。

見出し日付・カルーセル内容・「あと X 日」はすべてフォーカス日に追従する。
「今日」は JST 基準で算出する。

## 5. コンポーネント分割

`coding-style`（多数の小ファイル / 純関数の分離）に沿い、既存慣習どおり
**純ロジックは ESM モジュール `public/lib/schedule-view.js`** に、
**表示は babel JSX `public/screens-home.jsx`** に分ける。

純関数（`public/lib/schedule-view.js`・ユニットテスト対象）:
- `groupByDate(schedule)` — `date` 昇順 → 各日の試合を `time` 昇順に整形した
  `[{ date, matches }]` を返す。`date` 欠落要素は末尾の「日付未定」グループへ。
- `pickFocusDate(dateList, today)` — §4 のロジックでフォーカス日を返す。
- `formatMatchTeam(code)` — 3文字コード or スロット表記を「表示名 + 旗有無」へ正規化。
- `roundLabel(round)` — `A`→「グループA」、`R16`→「ベスト16」等の章ラベル。

表示コンポーネント:
- `<HomeScreen>` — データ取得 → `groupByDate` / `pickFocusDate` を呼び、子へ配る。
- `<MatchCarousel>` — フォーカス日の試合カルーセル（スワイプ/矢印/ドット状態を保持）。
- `<DayTimeline>` — 翌日以降の日付グループ一覧。
- `<MatchRow>` — タイムライン1行。

`index.html` のルーティング `'summary' → SummaryScreen`（462行）を `HomeScreen` に差し替える。

**注意**: `SummaryScreen` は部屋の比較画面（`screens-rooms.jsx:384` の `RoomCompareScreen`）でも
メンバーの予想サマリー表示に使われているため **削除しない**。ホームタブの描画先を切り替えるだけ。
`PodiumHero`/`MiniPick` も `SummaryScreen`・`InputScreen` が引き続き使うため温存する。

ESM 純関数はビルド慣習（`public/lib/*.js` を `export`、`index.html` の module script で
import → `Object.assign(window.WC, {...})` で babel 側へ橋渡し）に従って公開する。
チーム情報の参照は既存の `window.WC.TEAM[code]`（code→{ja,flag,c}）を使う。

再利用: `Card` / `Panel` / `Eyebrow` / `Flag` / `TeamLine` / `Icon` / テーマトークン
（`T.bg`/`T.card`/`T.line`/`T.text`/`T.sub`/`T.faint`/`T.accent`）。

## 6. エッジケース

- **ノックアウトのスロット表記**（チーム未確定）→ 旗の代わりにプレースホルダ枠 +
  「グループA 1位」「第73試合 勝者」等のラベル。確定後はコード/旗で表示。
- **`round` の種別** — グループ記号は「グループX」、ラウンド名はそのラベル（`roundLabel`）。
- **スケジュール空** → 「日程は準備中です」プレースホルダ。
- **カルーセルが1試合** → 矢印・ドットを隠す（破綻させない）。
- **全試合終了後** → 最後の試合日をフォーカス日として表示。

## 7. テスト

`testing` 方針に沿い、純関数のユニットテストを中心に置く:

- `groupByDate` — 日付/時刻ソート、同時刻試合、`date` 欠落要素の扱い。
- `pickFocusDate` — 当日試合あり / 当日試合なし→翌試合日 / 全試合終了後 の3境界。
- `roundLabel` / `formatMatchTeam` — グループ記号・ラウンド名・スロット表記の網羅。

表示コンポーネントは props 駆動の薄い層に保ち、ロジックを純関数へ寄せることで
テスト容易性を確保する。

## 8. 受け入れ条件

- ホームタブを開くと、フォーカス日の試合がカルーセルで表示される。
- カルーセルをスワイプ/矢印で当日の他試合に切り替えられる。
- カルーセル下に翌日以降の全日程が日付ごとに並び、スクロールで末尾まで見られる。
- 旧ホームコンテンツ（優勝サマリー等）が表示されない。
- ノックアウト未確定カード・空日程・大会後でもクラッシュせず妥当に表示される。
