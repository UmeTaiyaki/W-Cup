# 予想タブを「ホーム見た目＋編集ボタン」に刷新

日付: 2026-06-06

## 背景・目的

現在の予想タブ（`InputScreen`）は入力フォーム形式（縦並びの `Row` ＋ オプション予想の入口カード）。
ホームタブ（`SummaryScreen`）は見栄え重視の表示専用レイアウト（優勝の大カード `PodiumHero` ＋
準優勝・得点王の `MiniPick` ＋ オプション予想のインライン表示）。

予想タブをホームタブの見た目ベースに変え、各予想カードの右上に編集ボタンを置いて、その場から編集
できるようにする。これにより「自分の予想が完成形でどう見えるか」を見ながら編集できる。

ホームタブはそのまま残す（後日別コンテンツへ差し替え予定）。本対応ではホームタブの見た目は変えない。

## スコープ

- 対象: 予想タブ（`tab === 'input'`）の見た目刷新
- 非対象: ホームタブの内容変更、採点ロジック、データモデル、オプション編集画面そのものの変更

## 方針（実装アプローチ A: 共通部品の再利用）

見た目の定義を1箇所に保ち、ホームと予想で必ず一致させる。既存の編集フロー
（`TeamPicker` / `ScorerPicker` / `goOption`）をそのまま流用し、重複を作らない。

| 部品 | 置き場所 | 変更内容 |
|---|---|---|
| `PodiumHero` | `screens-core.jsx`（同ファイル内で共通化） | `onEdit?` を受け取り、有れば右上に編集ボタンを表示 |
| `MiniPick` | `screens-core.jsx`（同上） | `onEdit?` を受け取り、有れば右上に編集ボタンを表示 |
| `OptionViewScreen`（embedded） | `screens-optview.jsx` | `editable` ＋ `onEdit(id)` を追加。各セクション見出しの右に編集ボタン |
| `SummaryScreen` | `screens-core.jsx` | 共通化した部品を `onEdit` 無しで使う（見た目不変） |
| `InputScreen` | `screens-core.jsx` | body をホーム風カード＋編集ボタンに刷新。シート/削除は維持 |

`PodiumHero` / `MiniPick` は `SummaryScreen` 内のクロージャから、`T` / `pred` / `code` 等を props で
受け取る独立した関数コンポーネントへ切り出す（`screens-core.jsx` の同ファイル内に配置）。

## 予想タブ（input）の完成レイアウト

```
EDIT · ○○                              ← Eyebrow（共有ボタンは置かない）
予想を編集
─────────────────────────────────────
CHAMPION · 優勝              [✏️編集]    ← PodiumHero 右上に編集ボタン
 🏳️  日本                               → 優勝の TeamPicker を開く
─────────────────────────────────────
準優勝   [✏️]   │ 得点王   [✏️]          ← MiniPick 各右上に編集ボタン
 🏳️ 国名       │  選手名                 → 準優勝/得点王の Picker を開く
─────────────────────────────────────
［グループステージ / ノックアウト］      ← OptionView（embedded・editable）
📊 グループ順位 3/12組        [✏️編集]   → goOption('grouprank')
  …アコーディオン…
🥉 3位ワイルドカード          [✏️編集]   → goOption('thirdwild')
（ノックアウトタブ）          [✏️編集]   → goOption('knockout')
─────────────────────────────────────
○○を削除（non-solo のみ・現状維持）
```

## 編集インタラクション

- 基本予想（優勝 / 準優勝 / 得点王）: 編集ボタン → 既存のボトムシート
  - 優勝: `TeamPicker`（`setPick('champion', …)`）
  - 準優勝: `TeamPicker`（`exclude=[champion]`、`setPick('runnerUp', …)`）
  - 得点王: `ScorerPicker`（`setPick('topScorer', …)`）
- 未選択カードは編集ボタンとカード本体タップのどちらからでも開ける（ホーム見た目を保ちつつ未入力でも進める）
- オプション予想: 編集ボタン → 既存のフル編集画面へ遷移（`goOption('grouprank' | 'thirdwild' | 'knockout')`）

## 差分の扱い

- 共有ボタン: 予想タブには置かない（編集に集中。ホーム／オプション画面側に既存）
- みんなの優勝予想（`Everyone`）: 予想タブには出さない（編集タブのため）
- 参加者削除 UI: non-solo のときのみ表示、現状維持
- レスポンシブ: 既存の `wide` / モバイル分岐を踏襲（ホームと同じ振る舞い）

## エラーハンドリング / 後方互換

- データモデル・保存処理（`setPick` / `persist`）は不変。表示と編集導線のみの変更。
- `onEdit` 未指定時の `PodiumHero` / `MiniPick` は従来と同一表示（ホームタブの回帰を防ぐ）。
- `OptionViewScreen` の `editable` 未指定時は従来通り読み取り専用（ホーム／部屋での既存利用に影響なし）。

## テスト観点

- ホームタブ: 見た目・共有・みんなの予想が従来通り（回帰なし）
- 予想タブ: 各カードの編集ボタンから正しいシート／画面が開く。未選択カードも開ける。
- オプション編集ボタン: それぞれ対応するフル編集画面へ遷移し、戻ると予想タブに戻る
- 部屋（rooms）での `OptionViewScreen` embedded 利用が読み取り専用のまま壊れない
- モバイル／デスクトップ（wide）双方でレイアウト崩れがない
